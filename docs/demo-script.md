# Demo 劇本（對齊設計書 §23）

## 事前準備
1. `uvicorn app.main:app --reload`（backend/ 內）與 `npm run dev`（frontend/ 內）
2. 瀏覽器開 http://localhost:5173，以示範帳號「Vincent」登入

## Demo 1：語音建立案件
1. 首頁點「點擊並說出需求」進入對話頁
2. 按麥克風說：「我要預約明天下午洗兩台冷氣，地址是嘉義縣民雄鄉文隆村7鄰33號」
3. AI 只補問電話 → 說出「0912-345-678」
4. AI 顯示確認摘要 → 說「確認」
5. AI 回覆案件編號；回首頁可見「冷氣清潔／等待廠商確認」

## Demo 2：案件追蹤
1. 點擊剛建立的案件卡片
2. 展示：狀態徽章、表單資料、完整對話紀錄
3. 點「（Demo）模擬廠商確認」→ 狀態即時更新為「已確認」

## Demo 3：個人化長期記憶
1. 回首頁再建立新需求：「我想洗洗衣機」
2. 依序回答數量、類型、日期後，AI 主動詢問：
   - 「要安排在您偏好的下午時段嗎？」
   - 「要使用您上次的服務地址嗎？」
   - 「聯絡電話要使用上次留的⋯嗎？」
3. 全部回「好」，展示零重複輸入的體驗

## Demo 4：多使用者隔離
1. 登出，改以「美惠」登入
2. 首頁案件列表為空、AI 不會帶出 Vincent 的地址與偏好
